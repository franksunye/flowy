# 00_PROJECT_OVERVIEW - Flowy 项目概览

基于现有的 Flowy 流程图引擎，我们致力于将其发展为 AI/Agent 系统的核心 UI 组件。

---

## 🎯 项目愿景与定位

### 核心愿景
成为 **AI/Agent 生态系统的标准流程可视化引擎**，为自动化流程构建、展示与调试提供强大而易用的界面支持。

### 战略定位
- **极简易用**：保持轻量级特性，降低集成门槛
- **高度可扩展**：支持插件化架构，满足复杂场景需求
- **跨平台兼容**：Web/移动端全覆盖，支持主流前端框架
- **AI原生设计**：专为AI/Agent工作流优化的功能特性

---

## 📊 当前项目现状分析

### ✅ 已实现功能
- **基础拖拽连线功能**：完整实现，支持节点拖拽、自动连线、吸附等核心功能
- **JSON 数据导入导出**：完整的数据持久化机制，支持流程图的保存和加载
- **移动端触控支持**：已有基础的触控事件处理，支持移动设备操作
- **完整演示页面**：功能完备的 Demo，展示了自动化流程构建场景

### 🔄 待现代化改造
- **模块化架构重构**：当前为单文件结构（engine/flowy.js），需要拆分为模块
- **TypeScript 类型支持**：当前为纯 JavaScript，缺少类型安全保障
- **现代构建工具集成**：缺少 package.json、构建配置等现代开发工具链
- **单元测试覆盖**：完全缺失测试体系，需要建立完整的测试覆盖

### 🚀 待开发特性
- **React/Vue 适配器**：主流前端框架的组件封装
- **AI/Agent 专用功能**：节点参数配置、数据流管理、状态机控制
- **插件化机制**：支持自定义节点类型和行为扩展
- **可视化调试能力**：实时状态监控、日志追踪、错误诊断

---

## 📅 基于现状的分阶段发展路线

### 阶段一：现代化重构（1-2个月）🔄 急需开始
**目标**：将现有功能迁移到现代化架构
- 建立现代开发工具链（package.json、TypeScript、构建工具）
- 模块化重构：拆分核心引擎、渲染器、API接口
- 建立测试体系：单元测试、集成测试、E2E测试
- 完善文档：API文档、使用指南、贡献指南

### 阶段二：框架适配与优化（1-2个月）⏳ 待开始
**目标**：提供主流前端框架支持
- React 组件封装：hooks、props、事件处理
- Vue 组件封装：Composition API、响应式集成
- 移动端体验优化：手势支持、响应式布局
- 插件机制设计：节点注册、钩子系统、扩展API

### 阶段三：AI特性集成（2-3个月）⏳ 待开始
**目标**：专为AI/Agent场景优化
- 节点参数配置系统：输入输出端口、数据类型定义
- 流程状态管理：执行状态、数据流追踪、错误处理
- 可视化调试面板：实时监控、日志查看、性能分析
- AI框架集成：LangChain、OpenAI Function、自定义Agent支持

### 阶段四：生态建设（持续进行）⏳ 待开始
**目标**：建立开源社区生态
- 官方文档网站：完整的使用指南和API文档
- 示例项目库：各种AI/Agent应用场景的示例
- 社区运营：GitHub Issues、Discussions、贡献者指南
- 版本发布节奏：定期发布、向后兼容、升级指南

## 🛠️ 核心技术方向

### 1. 架构现代化
- **模块化设计**：Core Engine + Renderer + Adapters
- **TypeScript 重写**：完整的类型定义和类型安全
- **现代构建工具**：Vite/Rollup，支持多种输出格式
- **包管理**：NPM发布，语义化版本控制

### 2. 跨平台支持
- **框架适配器**：React、Vue、Angular组件封装
- **移动端优化**：触控手势、响应式设计、性能优化
- **浏览器兼容**：现代浏览器支持，渐进式增强

### 3. AI/Agent 特化功能
- **数据流管理**：节点间数据传递、类型验证、转换
- **状态机控制**：流程执行、暂停、恢复、回滚
- **调试工具**：可视化调试、性能监控、错误追踪
- **扩展机制**：自定义节点、插件系统、主题定制

### 4. 开发者体验
- **完整文档**：API参考、教程、最佳实践
- **开发工具**：CLI工具、模板生成、热重载
- **测试支持**：测试工具、Mock数据、自动化测试
- **社区支持**：示例代码、问题解答、贡献指南
